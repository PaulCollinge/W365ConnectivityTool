# Support

## How to file issues and get help

This project uses [GitHub Issues](https://github.com/PaulCollinge/W365ConnectivityTool/issues) to track bugs and feature requests. Please search the existing issues before filing new ones to avoid duplicates. For new issues, file your bug or feature request as a new issue.

When filing a bug, please include:

- The version of the tool (scanner exe version, or dashboard commit / date)
- The host environment (Windows 365 Cloud PC, Azure Virtual Desktop session host, physical Windows client, etc.)
- The Windows build (`winver`)
- A copy of the `W365ScanResults.json` if applicable (redact any sensitive endpoints first)
- Steps to reproduce and expected vs. actual behaviour

## Microsoft Support Policy

Support for this project is limited to the resources listed above. This is a community-maintained diagnostic tool and is provided on a best-effort basis. It is not covered by a Microsoft commercial support agreement.

For supported Windows 365 or Azure Virtual Desktop issues, please use your standard Microsoft support channels.
